Spring框架概述

Spring框架是Java企业级应用开发的事实标准，它提供了全面的基础设施支持，让开发者可以专注于业务逻辑的实现。

🎯 Spring核心理念

  • IoC（控制反转）：对象的创建和依赖关系由容器管理
  • AOP（面向切面）：将横切关注点从业务逻辑中分离
  • 声明式编程：通过配置而非编程实现功能
  • POJO编程：普通Java对象，无需继承特定类

🔧 IoC容器详解

IoC（Inversion of Control）是Spring框架的核心特性，它将对象的创建和依赖关系的管理交给Spring容器。

// 传统方式：手动创建对象和管理依赖
public class UserService {
    private UserDao userDao = new UserDaoImpl(); // 紧耦合
    
    public void save(User user) {
        userDao.save(user);
    }
}

// Spring IoC方式：依赖注入
@Service
public class UserService {
    @Autowired
    private UserDao userDao; // 松耦合，由Spring注入
    
    public void save(User user) {
        userDao.save(user);
    }
}

✂️ AOP面向切面编程

AOP允许将横切关注点（如日志、事务、安全）从业务逻辑中分离出来，提高代码的模块化程度。

// AOP切面示例
@Aspect
@Component
public class LoggingAspect {
    
    @Before("execution(* com.example.service.*.*(..))")
    public void logBefore(JoinPoint joinPoint) {
        System.out.println("执行方法：" + joinPoint.getSignature().getName());
    }
    
    @AfterReturning("execution(* com.example.service.*.*(..))")
    public void logAfterReturning() {
        System.out.println("方法执行成功");
    }
    
    @Around("@annotation(Transactional)")
    public Object handleTransaction(ProceedingJoinPoint point) throws Throwable {
        // 事务处理逻辑
        return point.proceed();
    }
}
